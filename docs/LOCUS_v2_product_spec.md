# LŌCUS v2 — Product Specification

> "던지기만 해. 나머지는 알아서."

## 1. 핵심 가치

사용자는 머릿속 무거운 것을 텍스트로 던지기만 한다.
태그, 카테고리, 우선순위를 정하지 않는다.
엔진이 자동으로 6가지 편의를 제공한다.

## 2. 6가지 실질 편의

### 2-1. 자동 분류
- 던진 텍스트를 기반(X)/성과(Y)/관계(Z)로 자동 분류
- 사용자가 보는 것: "일" "관계" "건강" 탭 또는 색상 구분
- 엔진: Weight 4축 (Domain, Intensity, Direction, Nature)

### 2-2. 미결 사항 추적
- "아직 끝나지 않은 것들" 리스트가 자동 생성
- Nature: unresolved인 항목을 자동 필터
- 사용자가 완료 처리하면 별이 희미해지거나 성운에 흡수

### 2-3. 반복 패턴 알림
- "이 고민 3번째입니다" — 같은 패턴의 반복을 감지
- Nature: recurring + 텍스트 유사도 기반
- 판단/조언 없이 사실만 전달

### 2-4. 통제 가능/불가 구분
- "이건 내가 할 수 있는 일" vs "이건 기다려야 하는 일"
- Direction: in/out 자동 분류
- 사용자가 보는 것: 두 영역으로 나뉜 뷰

### 2-5. 무게 순위
- "지금 가장 무거운 건 이거예요"
- Intensity × Nature × Direction 복합 mass로 자동 산출
- 사용자가 정하지 않아도 엔진이 무게를 읽음

### 2-6. 변화 리포트
- "지난주 대비 일 스트레스 줄고, 관계 고민 늘었어요"
- 주간/월간 Gravity 변화 추적
- 진단/조언 없이 사실의 변화만 전달

## 3. 사용자 경험 흐름

```
[무거운 것이 생김]
    ↓
앱 열기 → 텍스트 입력 → 던지기 (위로 스와이프)
    ↓
즉시: 별이 되어 성운으로 날아감 (시각 피드백)
자동: 분류 + 미결/반복/방향/무게 계산
    ↓
끝. 닫아도 됨.
    ↓
[다시 열면]
    ↓
- 자동 분류된 뷰 (일/관계/건강)
- 미결 리스트 ("아직 끝나지 않은 것들")
- 반복 알림 ("이 패턴 3번째")
- 통제 가능/불가 구분 뷰
- 무게 순위 ("가장 무거운 것")
- 변화 리포트 (주간/월간)
    ↓
3D 성운은 이 모든 것의 시각적 표현 (겉)
```

## 4. 겉과 속의 관계

| 겉 (사용자가 보는 것) | 속 (엔진이 하는 것) |
|---|---|
| 색상으로 구분된 별들 | Domain 분류 (X/Y/Z) |
| 밝은 별 = 무거운 것 | Intensity × Nature × Direction |
| 희미해지는 별 = 해결된 것 | unresolved → resolved 전환 |
| 반복 표시 (동일 궤도) | recurring 감지 |
| 안쪽/바깥쪽 위치 | Direction in/out |
| 성운 형태 변화 | 주간 Gravity 변화 |
| 항성 (가장 큰 별) | 가장 높은 mass |

## 5. 철학 유지

- 위로/힐링/진단/조언 금지
- 사실만 전달. "3번째 반복입니다" O, "쉬어야 합니다" X
- 스트릭/랭킹/알림 압박 없음
- 안 와도 성운은 조용히 변함 (죄책감 없음)
- 사용자가 스스로 발견하는 구조

## 6. 우선순위

### Phase 1 (즉시)
- [ ] 데이터 영속성 (localStorage → SQLite)
- [ ] 자동 분류 뷰 (기반/성과/관계 탭)
- [ ] 미결 사항 리스트
- [ ] 무게 순위

### Phase 2 (1~2주)
- [ ] 반복 패턴 감지 + 알림
- [ ] 통제 가능/불가 구분 뷰
- [ ] 던지기 제스처 개선 (스와이프)

### Phase 3 (2~4주)
- [ ] 주간/월간 변화 리포트
- [ ] Signal 고도화 (Gravity 기반)
- [ ] 공유 가능한 리포트 이미지 생성

## 7. 한 문장 정의

> LŌCUS는 머릿속 무거운 것을 던지면 알아서 정리해주는 공간이다.
> 태그도, 분류도, 우선순위도 직접 정하지 않는다.
> 던지기만 해. 나머지는 알아서 돼.
